2. SDK JS (recommandé)
Ajoute le script, initialise avec ta clé, puis envoie des events où tu veux dans ton app.
<script src="https://devconnect-dev.github.io/Nexus/nexus-sdk.js"></script>
<script>
  Nexus.init('nx_live_VOTRE_CLE');

  // Quand un évènement pertinent se produit dans ton app :
  Nexus.track('login_failed', {
    user_ref: 'user-42',
    severity: 'medium',
    details: { ip: '1.2.3.4' }
  });
</script>
3. Ou en HTTP direct (backend, curl...)
Aucun SDK requis : un simple POST JSON suffit, idéal depuis un serveur.
curl -X POST https://ekezdtageniidwjlgyys.supabase.co/functions/v1/nexus-ingest \
  -H "X-Nexus-Key: nx_live_VOTRE_CLE" \
  -H "Content-Type: application/json" \
  -d '{"event_type":"login_failed","user_ref":"user-42","severity":"medium","details":{"ip":"1.2.3.4"}}'
4. Référence rapide
event_type — chaîne libre définie par toi (ex : login_failed, signup, checkout_blocked, spam_detected). Reste cohérent d'un event à l'autre pour que les règles automatiques fonctionnent.
user_ref — optionnel, l'identifiant de l'utilisateur concerné dans ton système (peut être n'importe quelle chaîne).
severity — optionnel parmi info, medium, high, critical. Si omis, Nexus l'infère depuis le type d'event.
details — objet JSON libre (contexte, IP, payload...), utile pour l'investigation dans les Logs.

Chaque appel décrémente ton quota mensuel (voir plan ci-dessus). Au-delà, l'API répond 429 et l'event n'est pas enregistré.
